# Design Constraints — 从 ASCII 到美感的翻译层

> 目标：把 PRD 中的 ASCII 线稿和文字描述，"翻译"成 Pencil MCP 可执行的视觉约束规则，解决"模型努力但没美感"的问题。

---

## 0. 总则

**设计靠战术，不靠天赋**（Refactoring UI 核心观点）。每一条规则都是可执行的约束，不是主观评价。

**优先级**：当规则冲突时，按以下顺序裁决：
1. 可用性（Laws of UX）> 视觉美（CRAP + 9原则）> 个性表达
2. 如果不确定，选更简单的那个（Occam's Razor）

---

## 1. CRAP 四大原则（Robin Williams）

### 1.1 Contrast 对比

> 不同的东西就要截然不同，不要模棱两可。

**可执行约束**：

| 场景 | 规则 |
|------|------|
| 标题 vs 正文 | 字号差距 ≥ 2 倍（如 28px vs 14px）；字重不同（700 vs 400） |
| 主按钮 vs 次按钮 | 主按钮：填充色 `$1:--primary`；次按钮：`$1:--secondary` 或 outline 边框 |
| 可点击 vs 不可点击 | 可点击元素有视觉反馈（hover 态、阴影、下划线） |
| 当前选中项 | 用填充色或加粗区分，不靠文字颜色变化 |
| 危险操作 | 必须用 `$1:--destructive`，与常规操作颜色完全不同 |
| 禁用状态 | 透明度降至 40%（`opacity: 0.4`），不可用灰色文字代替 |

### 1.2 Repetition 重复

> 全篇的视觉元素必须一致，不随意发明新样式。

**可执行约束**：

| 元素 | 约束 |
|------|------|
| 字体族 | 全文仅用 1 种字体族（Inter），最多 1 种辅助字体 |
| 字重 | 全文仅用 3 种字重：400（正文）、500（小标题/标签）、700（大标题） |
| 字号阶梯 | 使用固定比例 1.25 阶梯：12 → 14 → 16 → 20 → 24 → 28 → 36 |
| 颜色 | 全局不超过 3 种主色，其余全部灰阶（neutrals） |
| 圆角 | 统一圆角值：按钮 6px，卡片 8px，模态 12px |
| 间距 | 使用 4 的倍数：4/8/12/16/24/32/48/64（见第 4 节） |
| 阴影 | 仅用 2 级：卡片（1 层浅阴影）、模态（2 层深阴影） |

### 1.3 Alignment 对齐

> 任何元素都不能随便放，必须与某个参考线对齐。

**可执行约束**：

| 规则 | 说明 |
|------|------|
| 左对齐优先 | 文字块默认左对齐，不居中（居中仅用于大标题/hero） |
| 同级元素对齐 | 同一行的元素顶部对齐（`alignItems: "start"` 或 `"center"`） |
| 缩进一致 | 所有列表项、表单字段使用相同缩进（如 16px 或 24px） |
| 不要居中正文 | 大段正文（> 2 行）必须左对齐，居中只用于单行标题 |
| 对齐参考线 | 页面级内容对齐左侧（侧边栏 + 主内容边界对齐） |

### 1.4 Proximity 亲密性

> 相关的东西靠近，无关的拉开。用留白代替分割线。

**可执行约束**：

| 关系 | 间距规则 |
|------|----------|
| 标题与所属内容 | 间距 8px（紧密） |
| 同组内容之间 | 间距 12-16px |
| 不同内容组之间 | 间距 24-32px |
| 大板块之间 | 间距 48-64px |
| 标签与其输入框 | 间距 4-6px |
| 表单字段之间 | 间距 16px |
| 表单组之间 | 间距 24px |

## 2. Visual Design 九大原则（IxDF）

### 2.1 Hierarchy 层级

> 最重要的元素最大/最粗/最醒目，次要的递减。

**可执行约束**：

| 层级 | 元素 | 字号 | 字重 | 颜色 |
|------|------|------|------|------|
| L1 | 页面主标题 | 28-36px | 700 | `$1:--foreground` |
| L2 | 区块标题 | 20-24px | 600 | `$1:--foreground` |
| L3 | 卡片标题 | 16-18px | 600 | `$1:--foreground` |
| L4 | 正文/描述 | 14px | 400 | `$1:--foreground` |
| L5 | 辅助文字 | 12-13px | 400 | `$1:--muted-foreground` |
| L6 | 提示/标签 | 12px | 500 | `$1:--muted-foreground` |

**规则**：每屏只有一个 L1 标题；L2 不超过 3 个。

### 2.2 Balance 平衡

> 视觉重量在画面中均匀分布。

**可执行约束**：
- 不要把所有内容堆在页面一侧
- 侧边栏宽度不超过页面的 1/4（280px / 1440px ≈ 19%）
- 指标卡片行：4 个等宽卡片，每个占 25%
- 空状态页面：内容居中，上下留白均衡

### 2.3 Scale 比例

> 元素大小遵循固定比例，不随意取值。

**可执行约束**：

| 元素 | 最小尺寸 | 推荐尺寸 |
|------|----------|----------|
| 可点击按钮 | 44×44px（Fitts's Law） | 44×44px 或 36×auto |
| 图标 | 16×16px | 20×20px 或 24×24px |
| 头像 | 32×32px | 40×40px |
| 卡片最小宽度 | 240px | 280-360px |
| 输入框高度 | 40px | 40px |
| 表格行高 | 44px | 48px |

### 2.4 Dominance 主导

> 每屏一个视觉焦点。

**可执行约束**：
- 主按钮每屏最多 1 个（CTA）
- 大标题每屏最多 1 个
- 不要出现两个竞争注意力的元素（如两个大按钮 + 一个大标题）

### 2.5 Similarity 相似

> 功能相同的元素看起来相同。

**可执行约束**：
- 所有删除/危险操作用同一种红色按钮
- 所有确认/提交操作使用同一种主色按钮
- 所有卡片用相同的圆角、阴影、padding

### 2.6 White Space 留白

> 留白是设计元素，不是"没放东西的地方"。

**可执行约束**：
- 页面内容区域至少留出 32px 内边距
- 卡片内部至少 24px padding
- 不要填满每一寸空间，留白占比 ≥ 30%
- 表单区域上下留白 ≥ 32px

### 2.7 Grid 网格

> 用栅格组织元素，不要自由漂浮。

**可执行约束**：
- 所有水平排列的元素使用 `layout: "horizontal"` + `gap`
- 卡片网格：`gap: 16px`，每个卡片 `width: "fill_container"`
- 表格列：固定列宽 + `fill_container`，不自由取值

## 3. Laws of UX — 可用性约束

### 3.1 Hick's Law — 选项越多决策越慢

**约束**：
- 主导航不超过 7 项（Miller's Law 同时生效）
- 下拉菜单项不超过 7 项
- 设置页面分组展示，不列出 20 个平铺选项
- 筛选器不超过 3-4 个维度

### 3.2 Fitts's Law — 目标越大越近越好点

**约束**：
- 所有可点击区域 ≥ 44×44px
- 重要按钮放在用户视线可达区域（不要藏到底部需要滚动）
- 移动端按钮更大（≥ 48×48px）

### 3.3 Jakob's Law — 遵循平台惯例

**约束**：
- 导航在顶部或左侧（不要右侧导航）
- 搜索框在右上角
- 表单提交按钮在右下方或底部居中
- 删除/危险操作放在右侧或底部，不要放在主流程路径上
- 模态框有关闭按钮（右上角 X）
- 错误提示在输入框下方，红色文字

### 3.4 Doherty Threshold — 反馈要在 400ms 内

**约束**：
- Loading 状态必须在操作后立刻出现
- 按钮点击有 immediate 视觉反馈（pressed 态）
- 表单验证错误实时显示，不提交后才提示

### 3.5 Aesthetic-Usability Effect — 好看 = 好用

**约束**：
- 视觉质量直接影响用户对可用性的判断
- 宁可功能简单但美观，不要功能全但难看
- 错误页面、空状态页也要设计，不能只放一行文字

## 4. Refactoring UI 战术速查

### 4.1 颜色

```
规则：选一个主色，其他全部灰阶。

主色：$1:--primary（按钮、链接、强调）
辅助色：$1:--secondary（次按钮、标签）
危险色：$1:--destructive（删除、错误）
其余：灰阶（$1:--foreground, $1:--muted-foreground, $1:--border）
```

**禁止**：
- 在一个页面用 3 种以上有彩色
- 用颜色区分不重要的信息层级
- 用纯黑 `#000000` 作为文字色（用 `$1:--foreground` 代替）

### 4.2 间距系统

```
4 的倍数：
4px   — 极小（图标与文字间距、badge padding）
8px   — 小（标签与输入框、按钮内部图标间距）
12px  — 中小（同组元素之间、表单标签间距）
16px  — 标准（卡片内容、表单字段之间、按钮组）
24px  — 大（卡片内部 padding、区块之间）
32px  — 更大（页面内容 padding、大区块之间）
48px  — 很大（页面级板块之间）
64px  — 极大（hero 与内容区之间）
```

### 4.3 阴影

```
Level 1（卡片）：1 层浅阴影，offset y=1, blur=1.75, color=#0000000d
Level 2（模态/下拉）：2 层阴影
  - y=2, blur=3.5, color=#0000000f
  - y=4, blur=5.25, color=#0000001a
```

**禁止**：
- 给输入框加阴影
- 给按钮加阴影（flat design）
- 给正文段落加阴影
- 阴影颜色不要用灰色（如 `#CCCCCC20`），用黑色带透明度

### 4.4 边框

```
规则：少用边框，用留白和背景色区分元素。

应该用边框的场景：
- 输入框（$1:--border, thickness: 1）
- 表格分隔线
- 卡片描边（仅在无阴影时）

不应用边框的场景：
- 两个卡片之间的分隔（用 gap/留白）
- 区块之间的分隔（用不同背景色或留白）
- 按钮（用背景色区分）
```

### 4.5 字体

```
主字体：Inter
字重：400（正文）、500（小标题/标签）、600（区块标题）、700（页面标题）
行高：1.428（body text, 14px）、1.5（card text, 16px）、1.4（modal title, 20px）
字母间距：默认，不调整（除非做全大写标签）
```

### 4.6 圆角

```
按钮：6px
输入框：6px
卡片：8px
模态/对话框：12px
头像/圆形元素：9999px（full pill）
徽章/标签：16px（半圆）
```

### 4.7 布局模板

**Dashboard 模板**：

│ Sidebar  │ Page Title                 │
│ 280px    │                            │

│ - Item 3 │                            │

## 5. 场景描述 → Pencil 翻译规则

> PRD 中通常没有 ASCII 图，更多是用户故事和场景描述。本节的翻译规则同时适用于 ASCII 和纯文字两种输入。

### 5.0 语义提取（文字描述 → UI 结构）

当 PRD 只有文字描述时，先提取 UI 结构再应用约束：

| 用户故事关键词 | 翻译为 UI 结构 | 约束规则 |
|---------------|---------------|---------|
| "查看列表" / "浏览" | Data Table | 用 `1:shadcnDataTable` 组件，含表头/分页 |
| "搜索" / "查找" | Search Input | 表格上方加搜索框，ref: `1:fEUdI` |
| "筛选" / "过滤" | Filter Dropdown / Tabs | 用 Tabs 或 Select 组件 |
| "添加" / "创建" | Form + Submit Button | 模态框或新页面，表单布局 |
| "编辑" | Form (预填充) | 同创建，字段有初始值 |
| "删除" | Destructive Button + Confirm | 红色按钮，点击出确认对话框 |
| "详情" / "查看" | Card 或 Detail Page | 用 Card Image 或纯文本卡片 |
| "空" / "暂无" / "还没有" | Empty State | 图标 + 提示文字 + CTA |
| "加载" / "等待" | Loading / Skeleton | 用 Progress 或骨架屏占位 |
| "通知" / "提醒" | Badge / Alert | 用 Badge 或 Alert 组件 |
| "分页" / "更多" | Pagination | 表格底部加 Pagination 组件 |
| "导航" / "切换页面" | Sidebar / Tabs | 侧边栏或顶部 Tabs |
| "确认" / "弹窗" | Dialog / Modal | 用 Dialog 或 Modal 组件 |
| "状态" / "进度" | Badge / Progress | 用 Badge 标签或 Progress 条 |
| "排行" / "统计" | Metric Cards + Table | 指标卡片 + 数据表格 |
| "设置" / "配置" | Form Groups + Tabs | 分组表单，用 Tabs 切换分类 |

**提取后**：得到 UI 结构（如 "Data Table + Search + Pagination"），再按 5.1-5.6 的规则生成 Pencil 原型。

### 5.1 标题翻译

```
输入（任意格式）: "Dashboard" 或 "# Dashboard" 或 页面标题叫"仪表盘"
→ Pencil: text, fontSize: 28, fontWeight: "700", fill: "$1:--foreground"
   下方间距: 24px
```

### 5.2 卡片翻译

```
输入: ASCII 卡片框 或 "用卡片展示项目信息"
→ Pencil: frame, ref: "1:pcGlv" (Card 组件)
```

### 5.3 按钮翻译

```
输入: [Submit] 或 "需要一个提交按钮"
→ Pencil: ref: "1:VSnC2" (Button/Default), descendants: {"1:8tnXG": {content: "Submit"}}
```

### 5.4 表格翻译

```
输入: ASCII 表格 或 "展示项目列表，含名称、状态、负责人"
→ Pencil: ref: "1:shadcnDataTable"
```

### 5.5 表单翻译

```
输入: ASCII 输入框 或 "用户需要填写邮箱和密码"
→ Pencil: ref: "1:1415a" (Input Group/Default)
```

### 5.6 导航翻译

```
输入: ASCII 导航列表 或 "左侧导航包含 Dashboard、Projects、Settings"
→ Pencil: ref: "1:qCCo8" / "1:jBcUh" (Sidebar Items)
```

### 5.7 空状态翻译

```
输入: "项目列表为空时提示用户创建"
→ Pencil: Card 容器内放图标(lucide:folder-open) + 文字("暂无项目") + 按钮("创建项目")
   布局: vertical, alignItems: "center", gap: 16, padding: 48
```

### 5.8 加载状态翻译

```
输入: "加载项目时显示进度"
→ Pencil: Progress 组件(ref: "1:hahxH") 或 骨架屏(灰色矩形占位)
   颜色: $1:--secondary (浅灰背景)
```

### 5.9 错误状态翻译

```
输入: "表单提交失败时提示"
→ Pencil: Alert/Destructive 组件(ref: "1:K53jF")
   位置: 表单顶部或输入框下方
   内容: 错误图标 + 错误信息文字
```

## 6. 质量检查清单

每次 Pencil 原型生成后，按以下清单自检：

### 视觉
- [ ] 全文字体族不超过 1 种
- [ ] 字号仅使用阶梯值（12/14/16/20/24/28/36）
- [ ] 字重仅使用 400/500/600/700
- [ ] 颜色不超过 3 种有彩色
- [ ] 没有硬编码 hex 颜色值（全用 `$1:--*` 变量）
- [ ] 圆角统一（按钮 6px、卡片 8px、模态 12px）

### 布局
- [ ] 所有文字左对齐（大标题除外）
- [ ] 同级元素间距一致
- [ ] 相关元素间距 < 不相关元素间距
- [ ] 留白占比 ≥ 30%
- [ ] 没有元素自由漂浮（都在 flex 容器内）

### 可用性
- [ ] 每屏只有一个 L1 标题
- [ ] 主按钮每屏最多 1 个
- [ ] 导航项 ≤ 7 个
- [ ] 可点击区域 ≥ 44×44px
- [ ] 危险操作用红色，与常规操作区分
- [ ] 有 Loading 状态定义
- [ ] 有空状态定义

### 交互
- [ ] 按钮有 hover/pressed 状态（组件自带）
- [ ] 表单有错误提示位置
- [ ] 模态框有关闭按钮
- [ ] 表格有分页

## 7. 常见错误与修正

| 错误 | 为什么不对 | 正确做法 |
|------|-----------|----------|
| 用边框区分卡片 | 边框让页面显得拥挤 | 用留白（gap: 16px）和阴影 |
| 正文居中 | 大段居中文字难以阅读 | 正文左对齐，居中仅用于标题 |
| 颜色太多 | 视觉噪音，没有重点 | 一个主色 + 灰阶 |
| 间距随机取值 | 不一致感 | 用 4 的倍数阶梯 |
| 字重随便用 | 层级混乱 | 只用 400/500/600/700 |
| 按钮大小不一 | 视觉不统一 | 统一高度 36px 或 44px |
| 没有空状态 | 真实场景缺失 | 每个列表/表格定义空状态 |
| 纯黑文字 | 太刺眼 | 用 `$1:--foreground`（通常 #0F0F0F 或 #18181B） |
