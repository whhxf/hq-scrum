# Design Constraints — UX/UI 专家规范

> 生效阶段：Phase 1（原型设计）作为决策参考 + Phase 4（代码实现）作为硬性约束。
>
> 角色：AI 的「UX 总监视角」。设计过程中自动加载并提示 PM 发现的问题，不是事后检查。
>
> 每一条规则都是二元的——要么遵守，要么违反。没有灰色地带。

---

## 原则 0：克制（Restraint）

Dieter Rams 第五条："好设计是克制的"（Good design is unobtrusive）。
在 UI 设计中，克制 = **做减法比做加法重要**。

### 规则

| 编号 | 约束 | 违反示例 |
|------|------|---------|
| C0.1 | 一个页面只有一种 primary action（最突出的操作） | 同时出现两个填充色主按钮 |
| C0.2 | 每屏只让用户思考一件事 | 空状态页同时展示"上传"和"功能介绍" |
| C0.3 | 如果一个元素可以被忽略而不影响核心流程，删掉它 | 非必要的 Badge、推荐链接、装饰性图标 |
| C0.4 | 不要为"可能有用的功能"预留空间 | 未定义用途的占位区域、过度通用的设置入口 |

### 验证方式
对每个设计元素问："没有它会怎样？"如果答案是"用户依然能完成核心任务"，删掉它。

## 原则 1：空间（Spacing）

Refactoring UI 核心观点：**空间是设计中最重要的元素，但被最严重低估。**

### 规则

| 编号 | 约束 | 数值 |
|------|------|------|
| S1.1 | 所有间距只取 4px 的整数倍 | 4, 8, 12, 16, 20, 24, 32, 48, 64 |
| S1.2 | 相关元素间距 ≤ 12px | 同一行内的图标和文字、标题和副标题、标签和输入框 |
| S1.3 | 不相关元素间距 ≥ 24px | 两个卡片之间、功能区域之间、表单组之间 |
| S1.4 | 容器 padding ≥ 内部元素 gap | 卡片 padding 24px 时，内部元素 gap ≤ 24px |
| S1.5 | 页面边缘安全区 ≥ 32px | 屏幕边缘到第一个内容元素的距离 |
| S1.6 | 同层级的多个元素间距必须一致 | 卡片网格的 gap 全部相同，不混用 12/16/20 |

### 间距速查表

| 值 | 用途 |
|----|------|
| 4px | 极小（图标与文字间距、badge 内部 padding） |
| 8px | 小（标签与输入框、按钮内图标间距、标题与所属内容） |
| 12px | 中小（同组元素之间、表单标签间距） |
| 16px | 标准（卡片内容、表单字段之间、按钮组、卡片网格 gap） |
| 24px | 大（卡片内部 padding、不同内容组之间、区块之间） |
| 32px | 更大（页面内容 padding、大区块之间） |
| 48px | 很大（页面级板块之间、空状态容器 padding） |
| 64px | 极大（hero 与内容区之间） |

### 在 Pencil 中的执行
- 使用 `gap: 8` / `gap: 16` / `gap: 24`，不使用任意值
- `padding: [12, 16]` 而不是 `padding: [13, 17]`
- 卡片高度用 `fit_content` + 内部 padding 撑开，不写死高度

## 原则 2：视觉层次（Hierarchy）

Refactoring UI 核心观点：**视觉层次不是靠颜色，是靠 size + weight + space 的组合。**

### 规则

| 编号 | 约束 | 说明 |
|------|------|------|
| H2.1 | 每屏只有一个视觉焦点 | 主按钮 > 次按钮 > 其他所有 |
| H2.2 | 文字字号只取阶梯值 | 11, 12, 13, 14, 16, 18, 20, 24, 28, 36 |
| H2.3 | 字重只使用 2 种：normal(400) 和 bold(600-700) | 不用 medium(500) 做层级区分 |
| H2.4 | 品牌色/主色只用于 primary action | 按钮、关键高亮，不用于大面积背景填充 |
| H2.5 | 次级操作用 outline 或纯文字 | 不造灰色填充按钮 |
| H2.6 | 非活跃内容用降低对比度 | 灰色文字 ≠ 删掉，保留信息但弱化 |

### 字号表

| 用途 | 字号 | 字重 | 颜色 |
|------|------|------|------|
| 页面大标题 | 28-36px | 700 | `$1:--foreground` |
| 区块标题 | 20-24px | 600-700 | `$1:--foreground` |
| 卡片标题 | 16-18px | 600 | `$1:--foreground` |
| 正文内容 | 14-16px | 400 | `$1:--foreground` |
| 辅助信息 | 12-13px | 400 | `$1:--muted-foreground` |
| 时间戳/标签 | 11-12px | 400 | `$1:--muted-foreground` |

**规则**：每屏只有一个大标题（28px+）；区块标题不超过 3 个。

## 原则 3：形状（Shape）

Refactoring UI 核心观点：**圆角是一种"功能性"设计，不是装饰。**

### 规则

| 编号 | 约束 | 数值 |
|------|------|------|
| R3.1 | 圆角只有 3 种：小(6px)、中(8px)、大(12px) | 不允许出现 7px、10px 等任意值 |
| R3.2 | 按钮圆角 = 小(6px) | 所有按钮、输入框统一 |
| R3.3 | 卡片圆角 = 中(8px) | 比按钮大一个级别，体现"容器"属性 |
| R3.4 | 模态/对话框圆角 = 大(12px) | 最大的圆角，体现"浮层"属性 |
| R3.5 | 纯圆形(9999px)只用于头像和全圆按钮 | 不用于卡片、面板等其他元素 |
| R3.6 | 徽章/标签圆角 = 16px | 半圆效果 |
| R3.7 | 边框粗细统一 1px | 不混用 2px、3px |
| R3.8 | 有阴影的元素不需要边框，有边框的元素不需要阴影 | 二选一，不叠加 |

## 原则 4：色彩（Color）

Dieter Rams：好设计是诚实的。色彩不是装饰，是信息的载体。

### 规则

| 编号 | 约束 | 色值 |
|------|------|------|
| C4.1 | 整个产品只有一个品牌色/主色 | `$1:--primary`（通常是蓝色或紫色系） |
| C4.2 | 品牌色只出现在两处：primary button 和关键高亮 | 不用于背景、不用于大面积填充 |
| C4.3 | 文字只用 3 级灰度 | `$1:--foreground`（主）、`$1:--muted-foreground`（次）、`$1:--muted`（辅助） |
| C4.4 | 背景只用 2 种 | `$1:--background`（页面）、`$1:--card`（卡片） |
| C4.5 | 边框只用 `$1:--border` | 不混用多个边框色值 |
| C4.6 | 危险操作用 `$1:--destructive` | 删除、错误提示 |
| C4.7 | 任何时刻屏幕上最多出现 2 种有彩色 | 品牌色 + 危险色，不出现红/绿/蓝/黄等其他颜色 |
| C4.8 | 成功状态用 `$1:--color-success` | 完成、通过、激活 |

**禁止**：
- 在一个页面用 3 种以上有彩色
- 用纯黑 `#000000` 作为文字色
- 用灰色 `#CCCCCC` 作为边框色

## 原则 5：留白（Whitespace）

Refactoring UI 核心观点：**留白不是"空"，是"呼吸"。**

### 规则

| 编号 | 约束 | 执行方式 |
|------|------|----------|
| W5.1 | 内容区域上下 padding ≥ 左右 padding | 卡片 `padding: [32, 24]` 而非 `[24, 24]` |
| W5.2 | 一个区域内的元素数量 ≤ 7 | 超过 7 个考虑分组、分页或折叠 |
| W5.3 | 列表元素之间必须有视觉分隔 | 卡片边框、gap 或背景色交替，不裸放 |
| W5.4 | 按钮的水平 padding ≥ 垂直 padding 的 2 倍 | 按钮高度 36px 时，宽度至少 80px |
| W5.5 | 空状态页的视觉重心在页面垂直居中 | 用 `justifyContent: "center"` + `alignItems: "center"` |
| W5.6 | 页面内容区域留白占比 ≥ 30% | 不要填满每一寸空间 |

## 原则 6：可用性（Usability）

Laws of UX 核心观点：**设计必须符合人的认知规律。**

### 规则

| 编号 | 约束 | 来源 |
|------|------|------|
| U6.1 | 主导航不超过 7 项 | Hick's Law + Miller's Law |
| U6.2 | 所有可点击区域 ≥ 44×44px | Fitts's Law |
| U6.3 | 导航在顶部或左侧，不在右侧 | Jakob's Law |
| U6.4 | 搜索框在右上角或顶部居中 | Jakob's Law |
| U6.5 | 表单提交按钮在右下方或底部居中 | Jakob's Law |
| U6.6 | 删除/危险操作不放在主流程路径上 | Jakob's Law |
| U6.7 | 模态框有关闭按钮（右上角 X） | Jakob's Law |
| U6.8 | 错误提示在输入框下方 | Jakob's Law |
| U6.9 | Loading 状态在操作后立刻出现 | Doherty Threshold |
| U6.10 | 每屏操作选项 ≤ 5 个 | Hick's Law |
| U6.11 | 筛选器不超过 3-4 个维度 | Choice Overload |

## 原则 7：反馈（Feedback）

Julie Zhuo：好设计让用户知道自己在哪、做了什么、接下来能做什么。

### 规则

| 编号 | 约束 | 实现方式 |
|------|------|----------|
| F7.1 | 当前选中项必须有明确视觉标识 | 填充色或加粗，不靠文字颜色变化 |
| F7.2 | 已完成和非活跃内容必须有对比 | 已完成用正常对比度，非活跃降低对比度 |
| F7.3 | 状态切换必须有视觉变化 | 播放/暂停、展开/收起图标切换 |
| F7.4 | 不要一次性展示所有信息 | 列表只显示 4-5 行，其余滚动或分页 |
| F7.5 | 每个列表/表格必须定义空状态 | 图标 + 提示文字 + CTA 按钮 |
| F7.6 | 每个加载操作必须定义 Loading 状态 | Progress 条或骨架屏占位 |

## 检查清单

Phase 1 设计阶段：每完成一个页面初稿，AI 自动逐项检查，提示 PM 发现的问题。
Phase 4 代码阶段：每次提交代码前，AI 自动检查实现是否与约束一致。

- [ ] C0.x 克制：没有多余的 Badge / Alert / 装饰元素
- [ ] S1.x 空间：所有间距是 4 的倍数（4/8/12/16/24/32/48/64）
- [ ] H2.x 层次：只有 1 个 primary action，字号只取阶梯值
- [ ] R3.x 形状：圆角只有 6/8/12/16/9999 五种
- [ ] C4.x 色彩：品牌色只出现 2 处，文字只用 3 级灰度
- [ ] W5.x 留白：内容区域上下 padding ≥ 左右，留白占比 ≥ 30%
- [ ] U6.x 可用性：导航 ≤ 7 项，点击区域 ≥ 44px
- [ ] F7.x 反馈：有选中态、空状态、Loading 状态

## Phase 4：Pencil → Code 翻译指南

每次将 .pen 原型翻译为代码时，对照此流程：

### Step 1：读取原型

用 `batch_get` 读取 prototype.pen 结构，用 `get_screenshot` 查看真实效果。

### Step 2：映射约束规则

| UI 元素 | 约束规则 | 代码实现 |
|---------|---------|---------|
| 容器（卡片、面板） | R3 + C4 + S1 | borderRadius + background + padding |
| 文字 | H2 + C4 | fontSize 阶梯值 + color 变量 |
| 按钮 | H2.1 + C4.1 + R3.2 | 品牌色按钮 / outline / 文字按钮 |
| 输入框 | R3.2 + C4.5 + S1.2 | Input 组件 + 1px 边框 |
| 列表 | S1.6 + W5.3 | flex/grid + gap 一致 |
| 导航 | U6.1 + U6.3 + S1.6 | Sidebar / Tabs 组件 |
| 标签/徽章 | S1.2 + C4.7 + R3.6 | Badge 组件 |
| 图标 | S1.2 | Lucide icon，16/20/24/48px |

### Step 3：生成代码

```
容器尺寸 = 内容宽度 + 左右 padding × 2
元素间距 = 关系紧密度映射（相关=8px，一般=16px，不相关=24px）
颜色值 = 从 tokens.css 变量引用，不硬编码
圆角 = 从 R3 规则查表（6/8/12/16/9999）
```

## 常见错误与修正

| 错误 | 违反规则 | 正确做法 |
|------|---------|----------|
| 用边框区分卡片 | R3.8 | 用 gap: 16px 和阴影 |
| 正文居中 | H2.6 | 正文左对齐，居中仅用于标题 |
| 颜色太多 | C4.7 | 一个主色 + 灰阶 |
| 间距随机取值 | S1.1 | 用 4 的倍数阶梯 |
| 字重随便用 | H2.3 | 只用 400 和 600-700 |
| 按钮大小不一 | S1.6 | 统一高度 36px 或 44px |
| 没有空状态 | F7.5 | 每个列表/表格定义空状态 |
| 纯黑文字 | C4.3 | 用 `$1:--foreground` |
| 圆角 7px/10px | R3.1 | 只用 6/8/12/16/9999 |
| 两个主按钮 | C0.1 | 一个 primary + 一个 outline |
| 导航 10 项 | U6.1 | 分组，不超过 7 项 |
| 按钮 30×30px | U6.2 | 最小 44×44px |
